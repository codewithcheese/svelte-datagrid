# How-to Guides

How-to guides are **task-oriented** recipes that guide you through the steps to solve specific problems. They assume you already have a working grid and want to accomplish something specific.

## Who are how-to guides for?

How-to guides are for developers who **know what they want to do** but need to know how. Unlike tutorials, they don't teach conceptsâ€”they show you how to complete a task.

## What makes a good how-to guide?

- **Focused**: Solves one specific problem
- **Practical**: Shows real, working code
- **Flexible**: Explains options and trade-offs
- **Complete**: Includes all necessary steps

## How to use these guides

1. Find the guide that matches your task
2. Follow the steps in order
3. Adapt the code to your specific needs

If you don't find a guide for your task, check the [Reference](../reference/index.md) for API documentation or [Explanation](../explanation/index.md) for conceptual understanding.
